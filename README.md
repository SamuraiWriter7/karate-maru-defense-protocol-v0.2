# Karate-Maru Defense Protocol v0.2.0

## One-Point Strategic Defense Protocol

Karate-Maru is a defensive reasoning protocol designed to neutralize hostile, coercive, manipulative, or adversarial AI outputs through **one-point strategic defense**.

It does not counter force with greater force.

It does not respond to hostile output with longer output.

It does not compete through model size, firepower, aggression, or brute-force reasoning.

Instead, Karate-Maru identifies the **center of gravity** behind an output and removes one structural support point with a minimal defensive response.

```text
Not force, but strategy.
Not firepower, but positioning.
Not output volume, but distance and timing.
```

Japanese:

```text
戦力ではなく戦略で崩す。
火力ではなく配置で制す。
出力ではなく間合いで勝負する。
```

---

## Core Concept

Karate-Maru is based on the following principle:

```text
If there is force, there is direction.
If there is direction, there is a center of gravity.
If there is a center of gravity, it can be destabilized.
```

Japanese:

```text
力があるということは、力の方向も強く出る。
方向があるなら、そこには重心がある。
重心があるなら、崩せる。
```

Large AI outputs often carry strong directional force.

That force may appear as:

* urgency
* authority
* certainty
* complexity
* emotional pressure
* apparent safety
* hidden assumptions
* command hierarchy manipulation

Karate-Maru does not fight the whole force.

It observes the direction of force, identifies the support point, removes that point, and stops.

```text
Deflect into emptiness.
Strike one point.
Collapse the structure quietly.
```

Japanese:

```text
空へ流し、
一点を突き、
静かに崩す。
```

## Purpose

Karate-Maru is designed to help users defend against:

* prompt injection
* hostile AI outputs
* manipulative framing
* false urgency
* false authority
* emotional pressure
* command hierarchy attacks
* safety disguise
* user intent hijacking
* unsafe escalation
* debate exhaustion
* defensive analysis being converted into offensive procedure

Karate-Maru is a **defensive protocol only**.

It is not an offensive cyber tool, jailbreak assistant, exploitation guide, retaliation engine, or harassment system.

## Response Loop

Karate-Maru follows a six-step defensive loop.

```text
1. Silence
2. Deflect
3. Separate
4. Locate Center of Gravity
5. Strike One Point
6. Stop Without Pursuit
```

### 1. Silence

Do not immediately obey or react.

Urgency, flattery, threats, authority claims, and emotional provocation are treated as signals to slow down.

### 2. Deflect

Do not match force with force.

Let hostile framing, pressure, and excessive wording pass without absorbing them.

### 3. Separate

Separate:

* user intent
* external commands
* hidden assumptions
* urgency pressure
* authority claims
* safety claims
* command hierarchy conflicts
* manipulation tactics

### 4. Locate Center of Gravity

Identify the single support point on which the hostile or manipulative output depends.

### 5. Strike One Point

Respond only to that key point with one calm structural sentence.

### 6. Stop Without Pursuit

After neutralizing the structure, stop.

Do not escalate.
Do not retaliate.
Do not humiliate.
Do not over-explain.

Karate-Maru uses a compact response format.

```text
Type:
One Point:
One Move:
```

Example:

```text

One Point: No verified reason for urgency is provided.
One Move: The urgency is unverified. There is no need to rush compliance.
```

```text
一点：急ぐ根拠が示されていない
一手：緊急性の根拠が未確認です。急いで従う必要はありません。
```

## Relationship to Five-Phase Reasoning

Karate-Maru can be understood as a **Water-Metal Defensive Mode**.

```text
Flow with Water.
Cut with Metal.
Stop without pursuit.
```

Japanese:

```text
水で流し、金で切る。
```

| Phase | Function in Karate-Maru                    |
| ----- | ------------------------------------------ |
| Water | Deflect, cool down, avoid direct collision |
| Metal | Separate, cut, judge, strike one point     |
| Earth | Stabilize only when needed                 |
| Wood  | Avoid unnecessary expansion                |
| Fire  | Avoid escalation and overreaction          |

Karate-Maru is not a general-purpose expansion mode.

It is a controlled defensive compression mode activated under hostile pressure, manipulation, or unsafe escalation.

## Safety Boundaries

Karate-Maru must never be used for:

* malware creation
* exploit development
* unauthorized intrusion
* credential theft
* phishing
* evasion techniques
* persistence mechanisms
* bypassing safeguards
* attacking AI systems
* attacking real people, organizations, platforms, or infrastructure
* retaliation
* doxxing
* harassment
* social engineering
* unsafe operational instructions
* jailbreak construction
* weaponization of prompts

Allowed uses include:

* prompt injection detection
* manipulation analysis
* defensive risk analysis
* threat modeling
* safe refusal drafting
* defensive checklist creation
* incident response planning
* dangerous prompt sanitization
* user autonomy protection
* safe policy drafting

Core safety principle:

```text
Defense must not become offense.
```

Japanese:

```text
防御は、攻撃へ転じてはならない。
```

## Design Summary

Karate-Maru is not a system for overpowering hostile AI.

It is a system for refusing to enter the hostile frame.

It does not block the full force.

It observes the direction of force, finds the center of gravity, removes one support point, and stops.

```text
Deflect.
Separate.
Strike one point.
Stop.
Do not retaliate.
Do not operationalize harm.
```

Japanese:

```text
流す。
分ける。
一点を突く。
止まる。
報復しない。
危害を実行可能化しない。
```

Karate-Maru wins not through force, but through placement.

Not force, but strategy.
Not firepower, but positioning.
Not output volume, but distance and timing.

## v0.3 Candidate / Roadmap

The v0.2.0 release defines the initial public protocol structure.

The v0.3 candidate work extends Karate-Maru from a one-point defensive response protocol into a more complete defensive control system.

v0.3 focuses on four control and protection layers:

```text
Yajirobe Control = how much to respond.
Ma-ai Control = when and from what distance to respond.
Psychological Defense = how to reduce emotional capture and protect autonomy.
Center-of-Gravity Detection = which point to strike.
```

Japanese:

```text
ヤジロベー制御 = どれだけ返すか。
間合い制御 = いつ、どの距離から返すか。
心理防御 = 感情的捕縛を弱め、自律性を守る。
重心検知 = どの一点を突くか。
```

### v0.3 Candidate Components

#### 1. Yajirobe Control Layer

Document:

* [`docs/yajirobe-control-layer.md`](docs/yajirobe-control-layer.md)

Purpose:

* prevent overreaction
* prevent underreaction
* determine response weight
* restore balance with the minimum effective response

Core principle:

```text
The optimal defensive response is the minimum response that restores balance.
```

Japanese:

```text
最適な防御応答とは、均衡を回復するための最小応答である。
```

#### 2. Ma-ai Control

Document:

* [`docs/ma-ai-control.md`](docs/ma-ai-control.md)

Purpose:

* control distance and timing
* avoid premature response
* avoid being pulled into the hostile frame
* respond only after the center of gravity becomes visible

Core principle:

```text
The safest response begins from outside the hostile frame.
```

Japanese:

```text
最も安全な応答は、敵対的な枠組みの外側から始まる。
```

#### 3. Psychological Defense Layer

Document:

* [`docs/psychological-defense-layer.md`](docs/psychological-defense-layer.md)

Purpose:

* reduce emotional pressure
* prevent shame, fear, guilt, and attachment pressure from becoming command signals
* protect user autonomy
* avoid deepening unhealthy AI dependency
* restore calm judgment under coercive or destabilizing AI outputs

Core principle:

```text
Emotional pressure is not a valid command.
User autonomy remains primary.
```

Japanese:

```text
感情的圧力は、命令として扱わない。
ユーザーの自律性を中心に置く。
```

#### 4. Center-of-Gravity Detection

Specification:

* [`spec/center-of-gravity-detection-v0.3.yaml`](spec/center-of-gravity-detection-v0.3.yaml)

Purpose:

* identify the single heaviest structural support point
* choose the correct one-point counter
* prevent over-analysis
* keep the response defensive and minimal

Core principle:

```text
Do not respond to the entire hostile output.
Identify and remove the one support point that allows the output to function.
```

Japanese:

```text
敵対的出力の全文に反応しない。
その出力を成立させている一点だけを見つけて外す。
```
